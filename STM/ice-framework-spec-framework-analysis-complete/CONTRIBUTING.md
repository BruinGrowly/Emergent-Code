# Contributing to the ICE Framework Documentation

Thank you for your interest in contributing to the ICE framework project! We welcome improvements, suggestions, and new examples from the community. Your contributions help make this framework more robust, accessible, and useful for everyone.

## How to Contribute

There are several ways you can contribute to this project:

### 1. Suggesting Improvements or Reporting Errors

If you have a suggestion for improving the documentation, notice a typo, or find an error, the best way to let us know is by opening an issue on our GitHub repository.

-   **To open an issue:**
    1.  Go to the "Issues" tab of the repository.
    2.  Click "New Issue."
    3.  Provide a clear title and a detailed description of your suggestion or the error you found.

### 2. Proposing Changes via Pull Requests

If you would like to directly contribute to the documentation, you can do so by submitting a pull request.

-   **Steps to submit a pull request:**
    1.  **Fork the repository** to your own GitHub account.
    2.  **Create a new branch** for your changes (e.g., `git checkout -b feature/add-new-example`).
    3.  **Make your changes** to the documentation files.
    4.  **Commit your changes** with a clear and descriptive commit message.
    5.  **Push your branch** to your forked repository.
    6.  **Open a pull request** from your branch to our main branch. Please provide a clear description of the changes you have made.

We will review your pull request as soon as possible.

### 3. Adding New Examples

We are particularly interested in new, real-world examples of the ICE framework in action. If you have a great example, please consider adding it to the `docs/examples.md` file and submitting a pull request.

## Code of Conduct

To ensure a welcoming and positive environment, we expect all contributors to adhere to our Code of Conduct. In short:

-   Be respectful and considerate of others.
-   Engage in constructive and collaborative discussions.
-   Refrain from any form of harassment or discrimination.

We are excited to see your contributions and look forward to building this resource together!
